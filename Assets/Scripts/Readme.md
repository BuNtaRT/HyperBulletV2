# Пуля - ша

первым делом у нас есть префаб на котором стоит дефолтное поведение, ```BulletBehavior```
по сути это мост между экземпляром поведения и игровыми событиями, у пули есть некоторое состояние в отдельном классе 
которое можно кидать при создании для обработкой эффектов, да это не правильно, за-то удобно с точки зрения обработки эффектов, что вы мне сделаете - это пет проект, 
и состояние используется для общения с экземпляром конкретного поведения

получается 3 слоя

1 - базовый клас прослойка который MonoBehaviour и обрабатывает события

2 - интерфейс поведения с реализацией конкретного выбранного поведения которое инициализируется

3 - состояние которое мы кидаем в поведение и в комбайнер эффектов для модификации

# Enemy spotted!!

тут очень похожая система на пулю, тоже 3 слоя где есть клас на префабе с интерактивностью, так же есть состояние, все тоже кидается в комбайнер эффетов
